Generated by repo scan on 2026-06-01. Last updated: 2026-06-03.

## Project purpose

This repository builds materials-science knowledge graphs from research papers. Main workflow:

1. Download or collect PDFs in `polymer_papers/`.
2. Extract schema-aligned terminology from PDFs with an LLM in `app/modules/extract_terms.py`.
3. Convert extracted terms JSON into MatKG graph JSON with `app/modules/json2kg.py`.
4. Query graph plus PDF evidence through KG-RAG in `app/modules/kg_rag_api.py`.
5. Evaluate generated graph files with scripts in `scripts/`.

## CBORG API call locations

CBORG is called through the OpenAI-compatible Python SDK.

### Primary implementation: `app/modules/extract_terms.py`

```python
class CBorgChatClient(ChatClient):
    def __init__(self, model: str, api_key=None, base_url=None):
        self.client = openai.OpenAI(
            api_key=api_key or os.environ.get("CBORG_API_KEY"),
            base_url=(base_url or os.environ.get("CBORG_BASE_URL") or "https://api.cborg.lbl.gov").rstrip("/"),
        )

    def chat(self, prompt, *, temperature=0.0, timeout=240):
        resp = self.client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=temperature,
            timeout=timeout,
        )
        return resp.choices[-1].message.content or ""
```

### Pipeline caller: `app/run_pipeline_cborg.py`

## Environment notes

`.env` drives all runtime configuration. `load_dotenv(override=True)` is used in all three entry points so `.env` always wins over stale shell env vars.

| Variable | Default | Notes |
| --- | --- | --- |
| `CBORG_API_KEY` | — | Required for CBORG backend. |
| `CBORG_BASE_URL` | `https://api.cborg.lbl.gov` | Public CBORG endpoint. Internal `api-local` requires VPN. |
| `KG_RAG_BACKEND` | `cborg` | `cborg` or `ollama`. |
| `KG_RAG_CBORG_MODEL` | `lbl/cborg-chat` | No `:latest` suffix — CBORG rejects it. |
| `KG_RAG_GRAPH` | `storage/kg/matkg_qwen3_235b_580papers.json` | KG file to load (15,815 nodes). |
| `KG_RAG_RETRIEVAL_BACKEND` | `lexical` | `lexical` avoids FAISS/Torch on Python 3.14. Use `semantic` when embedding stack is stable. |
| `KG_RAG_LLM_TIMEOUT` | `120` | Seconds before LLM call times out. |
| `KG_RAG_CTX_CHARS` | `6000` | Max context chars passed to LLM. Default 16k causes timeouts; 6000 is recommended balance. |
| `KG_RAG_SHOW_BASELINE` | `0` | Set to `1` to make a second non-RAG LLM call for comparison. |
| `PYSTOW_HOME` | `.cache/pystow` | Repo-local PyStow cache to avoid user path issues. |
| `MP_API_KEY` | — | Optional. Required for Materials Project formula cross-check. |
